If you see a "Get Pages site failed" error in your Actions tab, follow these exact steps:

1.  **Change Build Source**:
    - Go to your repo **Settings** > **Pages**.
    - Under **Build and deployment** > **Source**, change the dropdown from "Deploy from a branch" to **"GitHub Actions"**.

2.  **Enable Workflow Permissions**:
    - Go to **Settings** > **Actions** > **General**.
    - Scroll to **Workflow permissions**.
    - Select **"Read and write permissions"** and click **Save**.

3.  **Run Again**:
    - Go to the **Actions** tab.
    - Click on the failed "Deploy to GitHub Pages" workflow.
    - Click **"Re-run all jobs"**.
